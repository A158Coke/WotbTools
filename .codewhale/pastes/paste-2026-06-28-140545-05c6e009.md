# 任务：实现 Boost 功能后端 API

## 目标

为 WotBTools 新增 `boost / 寻找陪练` 功能的后端 API。

技术命名统一使用：

```text
boost
```

用户侧文案可以继续显示：

```text
寻找陪练
```

数据库表名使用：

```text
boost_request
booster_profile
boost_request_assignment
```

本任务只做后端，不做前端页面。

---

## 当前前提

假设 Flyway 已经创建了以下表：

```text
boost_request
booster_profile
boost_request_assignment
```

关系为：

```text
boost_request 1 -> N boost_request_assignment
booster_profile 1 -> N boost_request_assignment
```

一个需求可以有多个历史分配记录，但同一时间只能有一个 active assignment：

```sql
where unassigned_at is null
```

如果数据库已有 partial unique index：

```sql
create unique index uq_boost_request_active_assignment
on boost_request_assignment(request_id)
where unassigned_at is null;
```

后端也要配合这个约束处理并发/重复分配。

---

## 本次范围

需要实现：

```text
1. 玩家侧 boost request API
2. 管理员侧 boost request API
3. 管理员侧 booster API
4. 管理员分配/取消分配 API
5. 前端下拉选项 options API
6. Entity / Repository / Service / Controller / DTO
7. 基础 validation
8. 基础错误处理
9. Spring Security 权限控制
```

不要实现：

```text
支付
订单结算
自动匹配
公开打手市场
聊天
退款/仲裁
评价系统
打手自注册
账号密码收集
```

### POST /api/boost/requests

必须登录。

Request body：

```json
{
  "playerAccountId": 123456789,
  "playerNickname": "Coke_158",
  "region": "CN",
  "requestType": "COACHING",
  "targetDescription": "想找人指导中坦走位和残局处理。",
  "budgetRange": "可商量",
  "contactType": "QQ",
  "contactValue": "123456789",
  "availableTime": "中国时间晚上 20:00-23:00",
  "remark": "希望先沟通具体目标。"
}
```

Response：

```json
{
  "id": 42,
  "status": "NEW",
  "message": "需求已提交，管理员会人工审核并联系你。",
  "createdAt": "2026-06-28T12:30:00Z"
}
```

后端行为：

```text
1. 检查用户已登录
2. 校验字段
3. 拦截明显敏感信息
4. region 只允许 CN
5. 保存 boost_request
6. status 固定为 NEW
7. requester_user_id 从当前用户获取
8. 返回创建结果
```

前端不允许控制：

```text
status
adminNote
assignedBoosterId
requesterUserId
```

### PATCH /api/admin/boost/requests/{id}/status

必须 admin。

Request body：

```json
{
  "status": "REVIEWING",
  "adminNote": "已联系用户，等待确认目标。"
}
```

行为：

```text
1. 查找 request
2. 校验目标 status 是否合法
3. 更新 status
4. 如果 adminNote 不为 null，更新 adminNote
5. 更新 updatedAt
```

建议状态流转：

```text
NEW -> REVIEWING
NEW -> REJECTED
NEW -> CANCELLED

REVIEWING -> MATCHED
REVIEWING -> REJECTED
REVIEWING -> CANCELLED

MATCHED -> CLOSED
MATCHED -> CANCELLED

REJECTED -> REVIEWING
CANCELLED -> REVIEWING
```

MVP 可以不做严格状态机，但必须拒绝不存在的状态值。

Response：

```json
{
  "id": 42,
  "status": "REVIEWING",
  "statusLabel": "审核中",
  "adminNote": "已联系用户，等待确认目标。",
  "updatedAt": "2026-06-28T12:40:00Z"
}
```

## 管理员 assignment API

### POST /api/admin/boost/requests/{id}/assignments

必须 admin。

Request body：

```json
{
  "boosterId": 1,
  "note": "分配给擅长中坦的打手。"
}
```

后端行为：

```text
1. 查找 boost_request
2. 查找 booster_profile
3. 检查 booster.status == ACTIVE
4. 建议检查 booster.available == true
5. 检查该 request 当前没有 active assignment
6. 如果已有 active assignment，返回 409
7. 插入 boost_request_assignment
8. status = ASSIGNED
9. assigned_at = now()
10. unassigned_at = null
11. 更新 boost_request.status = MATCHED
12. 更新 boost_request.updated_at = now()
```

如果 booster 不可用：

```http
400 Bad Request
```

错误 code：

```text
BOOSTER_NOT_AVAILABLE
```

如果已经有 active assignment：

```http
409 Conflict
```

错误 code：

```text
ACTIVE_ASSIGNMENT_EXISTS
```

Response：

```json
{
  "id": 10,
  "requestId": 42,
  "booster": {
    "id": 1,
    "nickname": "TestBooster",
    "level": "ELITE",
    "levelLabel": "高手",
    "available": true,
    "status": "ACTIVE",
    "statusLabel": "正常"
  },
  "status": "ASSIGNED",
  "statusLabel": "已分配",
  "assignedAt": "2026-06-28T12:45:00Z",
  "unassignedAt": null,
  "note": "分配给擅长中坦的打手。",
  "createdAt": "2026-06-28T12:45:00Z",
  "updatedAt": "2026-06-28T12:45:00Z"
}
```

---

### PATCH /api/admin/boost/requests/{id}/assignments/current/unassign

必须 admin。

Request body：

```json
{
  "note": "用户要求更换打手。"
}
```

后端行为：

```text
1. 查找当前 active assignment
2. 如果没有 active assignment，返回 NO_ACTIVE_ASSIGNMENT
3. 设置 unassigned_at = now()
4. 设置 assignment.status = CANCELLED
5. 更新 note
6. 更新 assignment.updated_at
7. 如果 request.status == MATCHED，则改为 REVIEWING
8. 更新 request.updated_at
```

Response：

```json
{
  "id": 10,
  "requestId": 42,
  "status": "CANCELLED",
  "statusLabel": "已取消",
  "unassignedAt": "2026-06-28T13:00:00Z",
  "message": "当前分配已取消。"
}
```

## 管理员 booster API

### POST /api/admin/boost/boosters

必须 admin。

Request body：

```json
{
  "nickname": "TestBooster",
  "level": "ELITE",
  "available": true,
  "status": "ACTIVE",
  "contactType": "QQ",
  "contactValue": "987654321",
  "specialties": "中坦, 重坦, 排位",
  "description": "擅长中坦走位和高端局。"

```

校验：

```text
nickname 必填，最大 100
level 必填
available 如果 null，默认 true
status 如果 null，默认 ACTIVE
contactType 如果有值，只允许 QQ/WECHAT
contactValue 最大 255
specialties 最大 2000
description 最大 2000
```

## Validation 规则

### 敏感信息拦截

检查字段：

```text
targetDescription
remark
```

如果包含明显敏感词，拒绝：

```text
密码
验证码
账号密码
登录密码
password
verification code
```

返回：

```http
400 Bad Request
```

```json
{
  "code": "SENSITIVE_INFO_NOT_ALLOWED",
  "message": "请勿填写账号密码、验证码或其他敏感信息。"
}
```

MVP 简单 contains 即可，不需要复杂审核系统。

---

### region

Phase 1 只允许：

```text
CN
```

如果 region 为 null，可以默认设置为：

```text
CN
```

如果 region 不是 CN，返回：

```json
{
  "code": "INVALID_REGION",
  "message": "当前仅支持国服。"
}
```

---

### contactType

只允许：

```text
QQ
WECHAT
```

---

### requestType

只允许 `BoostRequestType` 里的值。

---

### booster level/status

只允许已定义值。

## 联系方式脱敏

玩家侧 DTO 返回 `contactValueMasked`。

示例：

```text
QQ 123456789 -> 123****789
WECHAT abcdefg -> abc***fg
短字符串 -> ***
```

管理员侧 DTO 返回完整 `contactValue`。

## Transaction 要求

以下操作必须加事务：

```text
创建 boost request
管理员更新 request status
创建 booster
更新 booster
分配 booster
取消分配 booster
```

尤其是分配 booster：

```text
1. 检查 active assignment
2. 插入 assignment
3. 更新 request status
```

必须在同一个事务中。

数据库 partial unique index 负责兜底防止并发重复分配。

如果插入 assignment 时触发唯一约束异常，转换成：

```text
ACTIVE_ASSIGNMENT_EXISTS
```

## 测试建议

按现有项目测试风格添加测试。

最低测试覆盖：

### 玩家创建 request

```text

非法 contactType 被拒绝
```

```text

```

```text

```

### 管理员 booster

```text

admin 可以更新 booster
admin 可以设置 available
非法 level/status 被拒绝
```

### assignment

```text

不能分配 banned/inactive booster
同一个 request 不能同时有两个 active assignment

如果 request.status 是 MATCHED，unassign 后改回 REVIEWING
```
